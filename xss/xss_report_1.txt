Use This Format to Write (in a text file like xss_report_1.txt)

1. Title:

> Stored Cross-Site Scripting (XSS) in Comment Section – example.com



2. Summary:

> The application fails to sanitize user input in the comment box. This allows an attacker to inject malicious JavaScript that runs when any user views the comment.



3. Steps to Reproduce:

1. Go to: example.com/article/123


2. In the comment box, type:

<script>alert("Hacked!")</script>


3. Submit the comment


4. Visit the same article again — the alert box appears



4. Impact:

> Can steal cookies, session tokens, deface UI, redirect users.



5. Recommendation:

> Escape all special HTML characters. Use frameworks that auto-sanitize (e.g., React, Angular) or apply server-side sanitization.



6. PoC:

> Screenshot or JS payload used
Payload:



<script>alert("Hacked!")</script>
